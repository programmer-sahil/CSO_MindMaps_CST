Computer System Organization

How a computer works internally: functional units, instruction execution, arithmetic, control, microprocessors, memory, and interfacing.

Unit 1 · Structure & Micro-operations
6 Hours · Basics

Functional view of a computer, Von Neumann model, data representation, and internal register-level operations.

▶ Computer Functional Units & Architecture
  • Input, Output, Memory, Arithmetic-Logic Unit (ALU), Control Unit (CU).
  • Von Neumann architecture: single memory for code & data, stored-program concept.
  • Bus structures: data, address, control buses; single-bus vs multiple-bus organization.
  • Basic operational concepts: instruction fetch, decode, execute cycle.
▶ Data Representation & Error Codes
  • Fixed-point representation: sign-and-magnitude, 1's complement, 2's complement.
  • Floating-point representation: sign, exponent, mantissa; range and precision idea.
  • Error detecting codes: parity bits, simple check codes (conceptual overview).
▶ Register Transfer & Micro-operations
  • Register transfer notation: R1 ← R2, conditional transfers, control signals.
  • Memory transfers: Read (MAR, MDR), Write cycles, bus-oriented transfers.
  • Arithmetic micro-operations: add, increment, decrement, clear, complement.
  • Logic micro-operations: AND, OR, XOR, NOT on register contents.
  • Shift micro-operations: logical, arithmetic, circular shifts.
  • Arithmetic-logic-shift unit (ALSU): combining arithmetic, logic, and shifting.

Unit 2 · Control, Arithmetic & Pipelining
20 Hours · Core Logic

How the control unit is built, how arithmetic is done in hardware, and how instructions are pipelined.

▶ Microprogrammed Control Unit
  • Control signals and the need for control unit.
  • Control memory: microinstructions and microprograms.
  • Address sequencing: next-address generation, branch in microprograms.
  • Design ideas of a microprogrammed control unit vs hardwired control (concept only).
▶ Computer Arithmetic
  • Addition & subtraction using 2’s complement adder/subtractor.
  • Multiplication algorithms: shift-and-add, Booth’s idea (concept).
  • Division algorithms: restoring / non-restoring division (concept).
  • Floating-point arithmetic: add, subtract, multiply, normalize, and rounding issues.
▶ Pipelining & Advanced Processing
  • Concept of pipeline: breaking computation into stages, throughput vs latency.
  • Arithmetic pipeline: example of pipelined floating-point operations.
  • Instruction pipeline: stages (Fetch, Decode, Execute, Memory, Write-back) and hazards.
  • RISC pipeline: simple fixed-length instructions, load-store design.
  • Vector processing: single instruction on multiple data elements.
  • Array processors: many processing elements working in parallel.

Unit 3 · Microprocessor Architecture
10 Hours · 8086 Focus

Generic instruction set ideas, then detailed view of the Intel 8086 microprocessor.

▶ Instruction Set & Design Principles
  • Instruction set: types of instructions (data transfer, arithmetic, logic, branch, I/O).
  • Programmer’s view: registers, memory model, addressing modes.
  • Design principles: simplicity, regularity, orthogonality, trade-off between hardware and software.
▶ Intel 8086 Architecture Overview
  • Block diagram: Bus Interface Unit (BIU) and Execution Unit (EU).
  • Pin functions: basic idea of control, address, and data pins.
  • Register structure: general purpose, segment, index, pointer, flag registers.
  • Segmentation: code, data, stack segments; logical vs physical address (segment:offset).
  • Interrupt mechanism: hardware & software interrupts, interrupt vector table concept.
  • Addressing modes: immediate, register, direct, indirect, indexed, based-indexed, with displacement.
  • Instruction groups: data transfer, arithmetic, logic, control transfer, string instructions.

Unit 4 · Assembly Language Programming
9 Hours · Coding

Writing, understanding, and organizing assembly programs for 8086-style processors.

▶ Basic Assembly Programs
  • Structure of an assembly program: data, code segments, labels.
  • Logical instructions: AND, OR, XOR, shift, rotate operations.
  • Branch instructions: conditional and unconditional jumps.
  • Call and return instructions: subroutine calls and stack usage.
▶ Algorithmic Examples
  • Sorting small arrays (e.g., bubble sort) using assembly loops.
  • Evaluating arithmetic expressions using registers and stack.
  • String manipulation: move, compare, scan, repeat prefixes.
▶ Directives, Procedures & Macros
  • Assembler directives: defining data, reserving storage, segment declarations.
  • Procedures: modularizing programs, passing parameters via stack or registers.
  • Macros: text substitution, writing and using simple macros.

Unit 5 · Memory & Digital Interfacing
15 Hours · System Level

Memory hierarchy, cache and virtual memory, and how the processor talks to RAM, ROM, and peripherals.

▶ Addressing & Memory Interfacing
  • Address lines and decoding: selecting memory chips using decoders.
  • Interfacing RAM: mapping blocks of addresses to RAM devices.
  • Interfacing ROM / EPROM: read-only storage for firmware, chip selection.
  • Timing and control signals for memory read/write cycles (concept level).
▶ Programmable Peripheral Interface
  • Need for parallel/serial I/O ports between CPU and external devices.
  • Programmable peripheral interface (PPI) concept: configurable I/O lines.
  • Basic idea of control word and I/O modes (no deep register detail required).
▶ Cache Memory
  • Motivation: bridging CPU–main memory speed gap.
  • Cache mapping techniques: direct-mapped, associative, set-associative (overview).
  • Hit and miss, hit ratio, miss penalty – effect on performance.
▶ Virtual Memory & TLB
  • Concept of virtual address vs physical address.
  • Page table: translating virtual pages to physical frames.
  • TLB (Translation Lookaside Buffer): speeding up address translation.
  • Demand paging idea and locality (concept only for exam).

Concise Definitions – CSO Quick Revision

Use these lines directly in short answers or as opening lines in 6-mark questions.
Functional Units: Logical blocks (Input, Output, Memory, ALU, Control) that cooperate to execute instructions in a computer.
Von Neumann Architecture: Stored-program model where instructions and data share the same memory and buses.
Bus: Common set of lines used to transfer data, addresses, or control signals between components.
Register Transfer: Operation that copies or moves data from one register to another under control signals.
Micro-operation: Simple operation performed on the data stored in registers (arithmetic, logic, or shift).
Fixed-Point Representation: Integer format where binary point is assumed at a fixed position in the number.
Floating-Point Representation: Real-number format using sign, exponent, and mantissa to represent a wide range of values.
Error Detecting Code: Extra bits added to data (e.g. parity) so that transmission or storage errors can be detected.
Control Unit: Part of CPU that generates control signals to sequence and synchronize all operations.
Microprogrammed Control: Technique where control signals are generated by executing microinstructions stored in control memory.
Pipeline: Arrangement of processing stages so that different parts of several instructions are executed in parallel.
Instruction Pipeline: Sequence of stages (fetch, decode, execute, etc.) used to overlap execution of multiple instructions.
RISC: Reduced Instruction Set Computer; uses a small, simple, and fast instruction set with load-store architecture.
Instruction Set: Complete collection of operations that a processor can execute.
Addressing Mode: Rule that specifies how to compute the effective address of an operand in an instruction.
Segmentation (8086): Memory management scheme that divides memory into logical segments like code, data, and stack.
Interrupt: Event that temporarily halts normal program flow so that the CPU can service an urgent request.
Assembly Language: Low-level language where each statement corresponds closely to a machine instruction.
Assembler Directive: Instruction to the assembler (not to the CPU) for defining data, segments, or other settings.
Procedure (Subroutine): Named block of instructions that performs a specific task and can be called from multiple places.
Macro: Named sequence of instructions expanded inline by the assembler during translation.
Address Decoding: Hardware process of using address lines to select a particular memory or I/O device.
Cache Memory: Small, fast memory placed between CPU and main memory to reduce average access time.
Hit Ratio: Fraction of memory accesses that are served by the cache; higher hit ratio means better performance.
Virtual Memory: Technique that gives an illusion of large main memory by using secondary storage as backing store.
Page Table: Data structure that maps virtual page numbers to physical frame numbers.
TLB (Translation Lookaside Buffer): Small associative memory that caches recent page-table entries to speed up address translation.
